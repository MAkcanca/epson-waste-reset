# epson-waste-reset

A single-file, **dependency-free** Python tool that reads and resets the
waste-ink counter on **1,271 Epson printer models** — so your printer keeps
working after you replace the waste pad yourself, with no per-reset fee.

It talks directly to a printer you own, using the printer's own maintenance
protocol. Every model constant comes from **already-published open-source
interoperability data** (see [Credits](#credits)); nothing is derived from any
commercial binary.

> **Not affiliated with Epson or any reset-utility vendor.** Community project,
> provided as-is. Resetting the counter does **not** empty the physical waste
> pad — replace or drain the pad first, or ink can overflow.

## Requirements

Python 3 — standard library only. No `pip install`, no net-snmp, no Docker.
(Rebuilding the model database with `tools/build_models.py` needs Python 3.11+
for `tomllib`; just *running* the tool works on any Python 3.)

## Quick start

Verify the tool before touching any printer (no printer required):

```
python reset_epson.py --self-test
python reset_epson.py --list-models      # all 1,271 supported models
```

### Over the network (recommended — model auto-detected)

Find the printer's IP (printer menu → Network Status), then:

```
python reset_epson.py --host 192.168.1.50 read       # detect model, show counter
python reset_epson.py --host 192.168.1.50 backup     # dump affected EEPROM to a file
python reset_epson.py --host 192.168.1.50 reset      # back up, confirm, then reset
```

Force a specific model with `--model "L3151"` if auto-detect can't match.

### Over USB (Windows — `--model` required)

```
python reset_epson.py --usb "EPSON L3150 Series" --model L3151 read
```

USB uses the already-installed Epson driver via the Windows print spooler (no
libusb, no driver swap). If it can't read back on your setup, connect the
printer to Wi-Fi and use `--host` — the network path is the reliable one.

## What `reset` does

1. Reads the current counter and **writes a backup file** of every affected
   EEPROM byte first.
2. Asks for confirmation (skip with `--yes`).
3. Writes that model's reset values.
4. Re-reads and shows before → after.

Then **power the printer off for 10 seconds and back on.**

## How it works

The counter lives in the printer's EEPROM, read and written through Epson's
"remote mode" control messages — carried as manufacturer-specific SNMPv1 `GET`
OIDs over the network, or the same control bytes over USB. `reset_epson.py`
hand-rolls the minimal SNMP/BER encoding in the standard library, so there are
no dependencies.

## Model database

`models.py` is generated by merging two open, independently reverse-engineered
databases. **104 models appear in both and their keys and reset maps agree
exactly (0 conflicts)** — that cross-validation is what makes the merge
trustworthy. 321 older models using a 1-byte-key command variant are recognized
but not yet enabled.

Both upstream source databases are **vendored under [`vendor/`](vendor/)**, so
the build is reproducible offline and survives the upstream URLs disappearing.
The generated `models.py` is committed too — the tool never needs the network.

```
python tools/build_models.py            # rebuild from the vendored sources (offline)
python tools/build_models.py --refresh  # re-download upstream, update vendor/, rebuild
```

## Safety

- Reads and backs up before any write.
- Requires explicit confirmation (`--yes` to skip).
- The printer validates the write key, so a malformed command is rejected
  rather than misapplied.
- `--self-test` verifies the protocol encoder and database invariants offline.

Keep the `backup-*.txt` files — they record the exact pre-reset EEPROM values.
